# 1.8.0 — Performance & Cleanup (M3): faster renders, a leaner image store, dead-code retirement, and a god-file split

## 🎬 Highlights

### The studio stops re-rendering itself to death (M3.1)

Typing in the prompt box or hovering a card used to re-render the entire
gallery grid. The MashupContext now hands out **stable callback / value
identities**, and `GalleryCard` + `PostReadyCard` are memoized with an
explicit image-passing handler contract — so a single card update no
longer cascades into a full-grid re-render. The bigger your gallery, the
more you feel it.

### Images live on disk, not in the settings blob (M3.2 / M3.2b)

The image store used to carry full data-URLs (the actual pixels) inside
the persisted record — a multi-megabyte JSON that had to be read and
rewritten on every change, and the real cause of the slow reloads. Pixels
now live as files on disk; the record keeps only a slim asset URL. The
**watermark image** got the same treatment (M3.2b): it moved out of the
settings object onto disk, so settings saves stay small.

### Dead code is gone (M3.3, P1–P3 a/b)

A round of retirement: dead docs and an unused dependency dropped (P1/P2),
the **nca** provider and the **mmx text path** removed (commit b), and the
default AI engine flipped to **vercel-ai** with the provider unions
narrowed (commit a). Shared prompt-building helpers were lifted into
`lib/ai-prompt-helpers.ts` so the studio route and the Director loop stop
duplicating logic.

### The god files start coming apart (M3.4-P4 / M3.3-P4)

`MainContent` and `SettingsModal` had grown into multi-thousand-line
files. This release extracts focused modules: per-view + scheduling
modules out of `MainContent`, the **Watermark** and **System Prompt**
editors out of `SettingsModal`, and pure helpers + shared types out of
`useImageGeneration`. Behavior is unchanged — these are structural splits
that make the next round of feature work tractable.

### Web Analytics (web build only)

Vercel Web Analytics is wired into the **web** build for performance and
usage insight. It is desktop-gated — the Tauri app ships without it.

### A pre-production bug batch

A 4-bug sweep landed alongside the refactors: prompt-commentary leakage,
the Higgsfield `--aspect_ratio` flag, weekly-fill counting, and
platform-sync. (Two of these — commentary leak and aspect-ratio — needed a
further, complete fix in 1.8.1.)

## 🔧 Breaking changes

none — the default AI engine flip is covered by a one-shot migration that
rewrites any stored `pi` / `nca` / `mmx` selection to `vercel-ai` on first
launch.

## 📋 Migration notes

No action required — auto-update applies everything on next launch. Your
existing images and watermark migrate to the new on-disk layout
automatically on first run.
