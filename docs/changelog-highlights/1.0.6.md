## 🎬 Highlights

### Node 25 + jsdom 29 fixed — tests are actually green now

The handoff claimed "1,243/1,243 tests pass." Reality was 1,199/1,243
with 44 failures across 6 test files. Root cause wasn't jsdom or
happy-dom or any test code — it was **Node 25's built-in `globalThis.localStorage`**.

Node 25 ships a `localStorage` global (a non-functional stub unless
`--localstorage-file=<path>` is passed). When the test environment
installed its own `localStorage` on globalThis, Node 25's stub won
because it was registered first. The result: `localStorage.clear
is not a function` on every jsdom/happy-dom test that touched it.

The fix is a small **Node 25 shim** at `tests/setup/node25-localstorage-shim.ts`
that installs a working in-memory `localStorage` on globalThis
*before* the test env initializes. The env then overwrites it with
its own. If it doesn't, tests still have a working fallback.

Result: **1,243/1,243 pass** in ~12s. The `--localstorage-file`
warning still appears in the output (it's Node 25's own noise, not a
test failure).

This shim is the recommended fix for any Node 25 + jsdom/happy-dom
project — not just MashupForge.

### ESLint is clean — 35 React 19 strict-rule fixes

The handoff claimed "~37 `as any` errors." Reality was 35 errors,
all React 19 strict-mode rule violations:

| Rule | Count | Fix |
|---|---|---|
| `react-hooks/set-state-in-effect` | 20 | Wrap setState in `queueMicrotask` (project convention from `HiggsfieldConnection.tsx`) |
| `react-hooks/refs` | 2 | Move ref-mirroring-state from render to `useEffect` |
| `react-hooks/purity` | 2 | Replace `useRef<number>(Date.now() + ...)` with `useState<number>(() => Date.now() + ...)`; derive `now` from state, not `Date.now()` in render |
| `react/no-unescaped-entities` | 3 | `'` → `&apos;` in JSX text |

**3 documented exceptions**: `KebabMenu.tsx` (2) and `CarouselApprovalCard.tsx`
(1) keep synchronous setState with an `eslint-disable-next-line
react-hooks/set-state-in-effect` and a justification comment. The
`queueMicrotask` deferred the setState past the test's `act()` block
and broke assertions on "menu items visible synchronously after the
kebab click" and "fire `onApprovePost` per sibling."
