# 2026-06-03 — MashupForge team bootstrap

## What landed

`.harness/` for MashupForge, set up to reuse Maurice's existing
global agent team (compounding model) rather than create
project-local shadows.

### Global agents filled in (persistent, compound across all of Maurice's projects)

- `dev` (BMAD Implement) — production code owner with cross-project rules
  (bun/npm, worktree discipline, AGPL awareness, security-by-default).
  The `coder` MiniMax agent stays on disk but we route to `dev`.
- `qa` (BMAD Measure) — test owner with cross-project policy
  (property-based for state machines, no jsdom, no flaky tests,
  crypto-tag-boundary awareness).
- `architect` (BMAD Architect) — design and solutioning with cross-project
  patterns (state machines, atomic-write guarantees, reconciler,
  dual-backend storage parity, peer-of-peer AI providers).

The other BMAD agents (`analyst`, `pm`, `ux`) and `verifier`
were left as-is — `verifier` is already a deep adversarial-review
specialist, and the BMAD roles have their phase-context already
in their descriptions.

### Project-local specialist reins (`.harness/reins/`)

3 reins extend the global team with MashupForge-specific deep
context. Each is BMAD-aware and falls back to its nearest global
role for non-specialist work:

- `post-lifecycle-expert` (extends `architect`) — the state machine,
  reconciler, IDB+SQLite storage, the v0.9.41 fix.
- `ai-providers-expert` (extends `dev`) — Higgsfield MCP, the
  text-model catalog, prompt engineering (SLCT / MCSLA / camera
  angles / anti-AI negative prompts).
- `tauri-desktop-expert` (extends `dev`) — Rust shell, webview
  boot, config.json, the Windows release pipeline.

3 redundant reins that were originally drafted (developer / tester
/ code-reviewer — shadows of global `dev` / `qa` / `verifier`)
were trashed.

### Project-level docs (`.harness/docs/`)

- `project-overview.md` — slim pointer to HANDOFF.md with the
  quick-reference card (stack, AI providers, pipeline, layout).
- `testing-policy.md` — MashupForge-specific addendum to the
  global `qa` agent (the 1,243-test bar, the framework, the

- `release-process.md` — full 5-step release flow with the

- `gotchas.md` — categorized list of all known gotchas (build,

  security).
- `v105-backlog.md` — pre-ranked v1.0.5 candidates from

### Orchestrator (`.harness/agent.md`)

Thin pointer to the global team + project HANDOFF. Knows the 7
global agents, the 3 reins, the release pipeline, the
worktree discipline, the token-rotation flag.

## Why this shape

Maurice's call: reuse the global team, level up where it fits,
keep project-local reins only for project-specific deep context.
The 3 specialist reins give the project the domain depth
(state machine, Higgsfield MCP, Tauri shell) without losing
the cross-project compounding that the global agents provide.

## What this means for the next agent

- Coding work → `dev` (not `coder`, which is MiniMax's).
- State machine / storage / reconciler work →
  `post-lifecycle-expert` rein (or `architect` if not in a
  rein session).
- Higgsfield / prompt engineering work → `ai-providers-expert`
  rein.
- Tauri / Rust work → `tauri-desktop-expert` rein.
- Test work → `qa`.
- Review / verify → `verifier`.
- Design / invariants → `architect`.
